Comments - The Structure:

The website follows a very easy structure. It consists of four main pages: Home, Past, Future and Comments. These page contains information unique to itself. Moreover, these pages are connected using a navigation bar styled as buttons across the top of each page. This navigation is consistent throughout each page, allowing the user to switch between each page in a smooth and intuitive manner without having to use the back arrows.

The website is stored in a home directory called Websystems. This directory contains the core files; Index.html, Past.html, Future.html and Comments.html. All of these HTMl pages share a common CSS stylesheet, which defines its visual design and layout. The Websystems directory also includes additional assets that contribute to the site's aesthetic. This includes the imported custom font VT323-Regular, which is used consistently across all pages for a retro look. The backgroung image bg.jpg provides visual consistency. Additionally, there's an icons folder within the directory that houses all the small PNGS that are scatteringly displayed around each web-page for decorative purposes.

Pages are divided into semantic sections using HTMLS tags. Each page on the site is built using a consistent layout which includes a header section (with a header and a navigation bar), a content section (which changes based on the topic of the page) and a footer. The header contains the navigation bar, which is made up of styled anchor (a) tags wrapped inside a container(div) which provides spacing and layout control. The main content is typically store within these div containers.

Structurally, the website's code relies heavily on div elements, along with the use of class, id, and span. These serve as "hooks" for CSS, allowing it to precisely target and style the pages. Furthermore, classes were prevalent throughout the code. These were used whenever a style was repeated across multiple pages, preventing the need to have to rewrite code over and over again.

Graphic Design Choices üé®

The graphic design aspects of the website were influenced significantly by my interests. My main aim was to create a website that was both visually engaging and reflected my personality. A consistent layout is used across all pages. Consisting of a header with navigation buttons, a content section, and a footer. This basic structure ensured clarity and familiarity for the user. I aimed for a retro aesthetic which I appealed to through the custom font¬†VT323-Regular, giving a typewriter like appearance reminiscent of early computing. The colour palette consisted of a soft brown text against a pastel yellow background which enhanced its readability while maintaining the warm aesthetic. Occasional use of other pastel tones further enhanced this aesthetic while minimising visual strain. The background image was obtained from the Studio Ghibli film "When Marnie Was There". This film followed similar aesthetics, maintaining a unified identity with the website while providing a consistent visual anchor for the viewer. Additionally, small illustrations such as those of Snoopy and other characters balance out the content and make it less overwhelming while adding to the decorative element. Overall, the design helps balance the functionality with a hint of personality, thus enhancing the user's experience by being both intuitive and aesthetically pleasant, using consistent styling and charming graphics to reflect the site's friendly tone.

Accessibility and Fallback design

To ensure accessibility for all users, especially those using screen readers or browing with images off, I used the alt attribute for all images across the site. This tag provides a descriptive alternative text to the image it is attached to, thus communicating the content or function of the image to the user if the image fails to load.

Furthermore, the website remains fully functional even after CSS is turned off. The content remains readable and navigable with all headings and paragraphs also remaining visible. This flow of information is preserved through the use of the HTMl semantic elements. However, without CSS, the visual and aesthetic appeal of the website decreases significantly. Hence, the website continues to adhere to the basic web access standards however it does lose its artistic quality. Without CSS, the content is presented linearly from top to bottom, with no control over the spacing, alignment or hierearchy beyond the basic HTML tags like h1 or p. Therefore, while the website does present all the required content and is navigable, the background images no longer appear, the layour reverts to block elements stacked vertically and any hover effects especially those applied on the navigation bar alongside the other spacing refinements are lost.

A description of the CSS Styles used:

I used a well organised CSS stylesheet to apply consistent and visually appealing styles across all pages. The table below explains each selector used, its type, what it styles and where its used:

body Element The body element applies the background image, sets the font and layout spacing of the webpage. It also applies the background image and adjusts it appropriately. Entire website
.button Class Styles the navigation links, buttons and their background colour and hover effects All pages
.top-banner, .banner Class Controls the white banner on top of each page. It gives it its colour and aligns it in the center of the page. All pages
#headings id This controls the heading box present on the Index.html page. It gives it its unique yellow colour and controls the headings present inside it. Index.html
.container Class It controls the main yellow backdrop in which all the smaller boxes storing the content are placed. It applies the margins, properly aligns it in the center of the page, gives it a solid border with a distinct colour and sets a width of 80%. All pages
.profile Class This class controls the layout of the second box within the page. It houses all the introdutory content about the page while giving it its colour, controlling its margins, padding and alignment. All pages

.extra Class This class controls and adjusts the second lot of boxes on the page. They make sure these boxes fit together and appear on the same level and maintains the spacing in between them. All pages
.box Class This class controls each box specifically, applying the unique colour scheme, padding, border-radius and width to them. All pages
.Other Class This class controls the layout and appearance of the last box. It ensures this box maintains the top and bottom margins while giving it its colour, padding and its border-radius. All pages
